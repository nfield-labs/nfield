## Responsibilities

### Maintainers are responsible for:

- Reviewing and merging pull requests (within 72 hours of submission)
- Triaging and responding to issues (within 48 hours)
- Publishing releases to PyPI
- Maintaining CI/CD pipeline health
- Enforcing the Code of Conduct
- Making architectural decisions per GOVERNANCE.md
- Keeping CHANGELOG.md up to date

### Maintainers are NOT expected to:

- Fix every bug themselves — that's what the community is for
- Be available 24/7 — this is a volunteer project
- Accept every feature request — scope is defined in GOVERNANCE.md

| [security@formatshield.dev](mailto:security@formatshield.dev) | Security vulnerabilities only — see SECURITY.md |

For general questions, use GitHub Discussions. Issues are for actionable bugs and confirmed feature requests only.

---

## Becoming a Maintainer

1. Contribute 3+ merged pull requests (earning Committer status)
2. Demonstrate sustained interest and care for the project's direction
3. Be nominated by an existing Maintainer
4. Receive no objections from existing Maintainers within 7 days

If you have 3+ merged PRs and are interested in becoming a Maintainer, open a GitHub Discussion to ask. We actively want to grow the maintainer team.
