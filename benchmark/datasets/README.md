# Benchmark datasets

Each real fixture is a self-contained directory under `real/<name>/`:

```
real/<name>/
  schema.json     target JSON Schema            (committed)
  gold.json       flat gold answer key          (committed; absent => coverage-only)
  document.txt    source document               (committed if small; W&P is gitignored)
```

| Fixture | Gold fields (N) | Scorable | Notes |
|---|---|---|---|
| `clinicaltrial` | 304 | yes | ClinicalTrials.gov record, typed/nested |
| `factbook_us` | 335 | yes | CIA World Factbook, single country |
| `factbook_multi` | 1045 | yes | Factbook, multi-country (the N≈1000 anchor) |
| `war_and_peace` | — | no (coverage-only) | 3.3 MB Gutenberg text; document fetched on demand |

The gold answer keys correspond to the exact committed document snapshot — that
pairing is what makes a Value Accuracy number reproducible. The small documents
are committed for that reason; the large coverage-only War & Peace text is
regenerated by `gen/` rather than committed.

`gen/` holds the generators that produced these fixtures (provenance and
regeneration). They fetch from public sources (ClinicalTrials.gov, the CIA World
Factbook, Project Gutenberg) and are run manually, not in CI.

The registry that maps a name to its context budget and loader lives one level
up in `benchmark/datasets.py`.
